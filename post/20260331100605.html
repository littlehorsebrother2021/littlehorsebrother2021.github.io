SMPS:SMPS step-down converter（开关电源降压转换器）,是STM32N6芯片自带的一个开关电源降压转换器，这个设施的启用与否取决于硬件设施设计时是否对外部供电存在处理，能否让核心在没有转换器的情况下正常工作。我所使用的开发板并没有，因此这里需要将其变成“PWR_SMPS_SUPPLY”，完整的配置图如下：

这里需要注意的是，由于N6的一个工程可能会包含FSBL和Application两部分，因此生成的工程目录中和正常的F1,F4,H7目录不一样，这里给出具体解释：

STM32N6 工程真实目录结构说明 (基于 STM32CubeIDE)

本项目基于 STM32N6 系列微处理器开发。STM32CubeMX 在生成 STM32CubeIDE 工程时，采用了”源码与 IDE 工作区严格分离“的设计逻辑。

以下是本工程真实生成的物理目录层级及其核心功能详解：

🗂️ 整体物理目录树预览
📦 Project_Name (工程根目录)
├── 📄 Project_Name.ioc               # CubeMX 图形化核心配置文件
├── 📄 .mxproject                     # CubeMX 内部生成状态记录文件
│
├── 📁 Appli                          # 【纯源码区】应用核心代码 (不含任何 IDE 配置)
│   └── 📁 Core                       # 包含 main.c, 头文件及外设初始化 C 代码
│
├── 📁 Drivers                        # 【纯源码区】官方底层驱动库
│   ├── 📁 CMSIS                      # Cortex-M55 内核文件及启动汇编
│   └── 📁 STM32N6xx_HAL_Driver       # STM32 HAL/LL 库源码
│
├── 📁 Secure_nsclib                  # 【安全特性】TrustZone 相关的安全/非安全域接口存根 
│
└── 📁 STM32CubeIDE
    │   # 【IDE 工作区】所有的编译、链接和工程配置文件全在这里！
    ├── 📄 .project                   # 整个 Eclipse 工作区的定义文件
    └── 📁 Appli                      # 针对 Appli 应用的独立子工程文件夹
        ├── 📄 .cproject & .project   # CubeIDE/Eclipse 的核心工程配置文件
        ├── 📄 Project_Appli Debug.launch # 点击“Debug/Run”时调用的启动/下载脚本
        ├── 📁 Application  
        │   # IDE 内部虚拟映射的源码文件夹 (指向外层的 Appli/Core)
        ├── 📁 Drivers                # IDE 内部虚拟映射的驱动文件夹 (指向外层的 Drivers)
        ├── 📁 Debug                  # 编译输出目录 (包含生成的 .elf, .bin, .map 等文件)
        ├── 📁 Release                # 编译输出目录 (包含生成的 .elf, .bin, .map 等文件)
        └── 📄 STM32N647X0HXQ_*.ld    # 【关键】所有的链接脚本文件都在这里！

2.6 添加自制文件夹

 由于后续开发需要添加非常多的驱动文件，因此这里提前说明如何添加自制文件夹/文件。

  1. 在工程根目录文件夹Drivers下新建一个文件夹BSP

  2. 在IDE中右键点击工程目录Drivers，选择New->Folder，

  3. 在弹出的窗口中输入文件夹名称，不要立刻点击Finish，而是先点下面的Advanced >> (高级) 按钮。

  4. 展开高级选项后，你会看到下面多出了一块区域。勾选 Link to alternate location (Linked Folder)（链接到备用位置/链接文件夹）。

  5. 这时下面的路径框亮起了。点击旁边的 Browse… (浏览) 按钮。

  6. 在弹出的窗口中，选择你刚刚在工程根目录Drivers文件夹下新建的文件夹BSP，然后点击 OK。

  7. 最后，点击窗口右下角的 Finish。
     这样，你就在工程目录Drivers下添加了一个名为BSP的文件夹，并且这个文件夹的链接位置是工程根目录Drivers下的BSP文件夹。以后，你在这个文件夹下添加的文件，都会被链接到工程目录Drivers下的BSP文件夹中。
    此时的工程结构如下：

📦 Project_Name (工程根目录)
├── 📄 Project_Name.ioc               # CubeMX 图形化核心配置文件
├── 📄 .mxproject                     # CubeMX 内部生成状态记录文件
│
├── 📁 Appli                          # 无变化 
├── 📁 Drivers                        # 【纯源码区】官方底层驱动库
│   ├── 📁 CMSIS                      # 无变化
│   ├── 📁 BSP                        # 【自制区】自制驱动库
│   └── 📁 STM32N6xx_HAL_Driver       # STM32 HAL/LL 库源码
│   
├── 📁 Secure_nsclib                  # 无变化
│
└── 📁 STM32CubeIDE                   # 无变化
    ├── 📄 .project                   # 无变化
    └── 📁 Appli                      # 无变化
        ├── 📄 .cproject & .project   # 无变化
        ├── 📄 Project_Appli Debug.launch # 无变化
        ├── 📁 Application            # 无变化
        ├── 📁 Drivers                
        │   # IDE 内部虚拟映射的驱动文件夹 (指向外层的 Drivers, BSP在其中)
        ├── 📁 Debug                  # 无变化
        ├── 📁 Release                # 无变化
        └── 📄 STM32N647X0HXQ_*.ld    # 无变化

 另有,如果需要将某个文件夹单独列出方便调用，可以进行以下操作：

  1. 点击左侧工程树所选工程的总目录Project_Name_Appli,右键选择Properties->C/C++ General->Path and Symbols-> Source Location选项卡。
  2. 点击右侧的 Add Folder (添加文件夹) 按钮，
  3. (举例)在弹出的目录树里，依次展开：你的工程 Project_Name_Appli -> Drivers ->BSP-> 选中 UART 文件夹后->点击OK
  4. 点击Apply and Close，你会看到原先在Drivers/BSP 文件夹下 的UART已经被提升到了与Drivers同级的位置。

2.7 添加自制文件 & 访问路径

 在工程根目录Drivers中的文件夹BSP添加相应代码文件，然后在IDE中右键点击工程右键flash,即可刷新进入到工程中。

 接下来添加文件路径，由于BSP后续内容较多，因此索引路径只有一条/${ProjName}/Drivers/BSP。操作如下：

  1. 点击左侧工程树所选工程的总目录Project_Name_Appli,右键选择Properties->C/C++ General->Path and Symbols-> Includes选项卡。
  2. 点击右侧的 Add... (添加…) 按钮。
  3. ⚠️ 核心警告：绝对不要在 Directory 文本框里手动打字！
  4. 点击输入框下面的 Workspace... (工作空间…) 按钮。
  5. 在弹出的目录树里，依次展开：你的工程 Project_Name_Appli -> Drivers -> 选中 BSP 文件夹。
  6. 点击 OK。此时，文本框里会自动填入 /${ProjName}/Drivers/BSP。
  7. 仔细看输入框的下方，确保勾选了 Is a workspace path（这是一个工作空间路径）。 （通常用 Workspace 按钮选出来的，这个勾会自动打上，这就是纯手打无法触发的灵魂所在！）
  8. 点击 OK。

3.添加代码

 在Drivers/BSP文件夹下新建一个文件夹UART，然后在UART文件夹下新建一个文件uart.c，在uart.c文件中添加以下代码：

/**
 ****************************************************************************************************
 * @file        uart.c
 * @version     V1.0
 * @brief       串口驱动代码
 ****************************************************************************************************
 */
#include "uart.h"

/**
 * @brief   重定向C库的printf函数到串口
 * @param   无
 * @retval  无
 */
int __io_putchar(int ch)
{
    HAL_UART_Transmit(&huart1, (uint8_t *)&ch, 1, 0xFFFF);

    return ch;
}


/**
 * @brief   重定向C库的scanf函数到串口
 * @param   无
 * @retval  无
 */
int __io_getchar(int ch)
{
    HAL_UART_Receive(&huart1, (uint8_t *)&ch, 1, 0xFFFF);

    return ch;
}

 在Drivers/BSP文件夹下新建一个文件夹UART，然后在UART文件夹下新建一个文件uart.h，在uart.h文件中添加以下代码：

/**
 ****************************************************************************************************
 * @file        uart.h
 * @version     V1.0
 * @brief       串口驱动代码
 *
 ****************************************************************************************************
 */

#ifndef __UART_H
#define __UART_H

#include "main.h"
#include <stdio.h>
#include "usart.h" //只有当你勾选了`Generate peripheral initialization as a pair of '.c/.h' files per peripheral`时才需要包含这个文件




#endif

...

/* USER CODE END Includes */

4.关于在工程上复制和创新

 在工程上复制和创新，原本在keil5 上是可以直接复制文件夹改名字就可以了，但是STM32CUBEIDE上他会针对你的工程文件进行独立的签名，这种签名非常多，如果手动改的话将会非常繁琐，并且每个加入cubeide的工程之间还不能重名，否则无法添加进cubeide，因此我更建议如果要在原工程上做开发，请直接只复制一个cubemx文件，即xxx.ioc然后将xxx更名为你需要的名字比如zzz，然后打开zzz.ioc文件，在新的文件夹下重新生成工程，最后直接在cubeide上添加文件，这样就不会出现重名的问题了。

 通过以上步骤，我们成功在STM32CubeIDE上添加了自制驱动文件，并实现了串口打印功能。在后续的开发中，我们可以根据需要添加更多的自制驱动文件，以实现更多的功能。这篇文章是STM32N6 从0到1开发的巨大里程碑，后续文章将会继续介绍STM32N6的更多功能。
